# 计算器使用说明

<font color = "red">注：使用前请先提前下载Eigen库，否则会出现部分功能无法使用甚至编译错误的情况。</font></br>  

## 当前已经实现的功能
1. [多项式计算](#mutiple)</br> 
多项式加法、多项式减法、多项式乘法，任意阶积分，任意阶微分等。

2. [四则运算](#calculate)</br>
整数与浮点数的加法、减法、乘法、除法、幂运算等等。
  
3. [矩阵运算](#matrix)</br>
矩阵的加法，矩阵的减法，矩阵的乘法，矩阵的转置，矩阵求行列式，矩阵求特征值。
  
4. [一元表达式运算](#calculx)</br>
任意给出含一元变量的表达式，并给出一元变量的赋值，即可运算给出结果。仅支持符合c++命名规则的变量名。
  
5. [一元函数运算](#function)</br>
支持函数定义，函数运行，在变量名完全统一的情况下的函数互调。
</br>

## 计算器使用说明

<a id = "mutiple">

### 一元多项式
在选择模式时在命令框中输入‘1’，然后即可进入一元多项式模式。</br>
然后会询问您“要输入的项数”。此时您需要了解您多项式的最大的次数项，并输入<font color = "red">此次数+1</font>对应的值。也即即使不存在的项也要视为系数为零的存在项。</br>
然后会让您按照从小到大的顺序输入<font color = "red" size = 4><strong>系数</strong></font>，<font color = "red">即使为零也要输入‘0’</font>。</br>
接下来会询问您是否需要输入第二个表达式，如果仅做加减乘除则<font color = "red">不必要</font>输入，如果仍然输入则微积分只会对第一个输入的表达式运算。输入方式同上。</br>
运算后的结果会作为第一个表达式存储，如果要继续运算可以直接重复以上步骤，否则如果要退出，按照指引输入‘#’即可。</br>
</a>
<a id = "calculate">

### 四则运算

在选择模式时在命令框中选择‘2’，然后即可进入四则运算模式。</br>
然后会提示您输入要计算的四则运算表达式，注意：输入的时候<font color = "red">不要带空格</font>，不要出现数字和运算符号之外的符号，否则会判定运行异常而退出。</br>
输入完成后按回车（enter键）即可出现结果。结果不会保存，且会直接退出四则运算模式。</br>
</a>

<a id = "matrix">

### 矩阵运算
在模式选择状态下输入‘3’，即可进入矩阵运算模式。</br>
首先输入第一个矩阵，分别输入行数和列数，两个数字用空格隔开。然后按行输入矩阵中的数据，数据间用空格或回车分割。
然后会询问是否输入第二个矩阵，如果需要输入1，否则输入2。，只有加减乘需要输入第二个矩阵，其他的不需要。如果不需要而输入了，则默认操作第一次输入的矩阵。</br>
<font color = "red">注意：矩阵运算为单次运算，运算结束就会退到模式选择栏。</font></br>
而后会询问您要做的运算种类，加法输入‘+’，减法输入‘-’，乘法输入‘*’，转置输入‘/’，行列式输入‘=’，<font color = "red">如果运算行列式，应先将第一行的注引号去掉</font>。
</a>

<a id = "calculx">

### 一元表达式运算
<font color = "red">注意：变量命名应遵循c++的命名规则，否则会出现运行异常甚至结果错误的问题。</font></br>
在模式选择时在命令框中输入‘4’，即可进入一元表达式模式。</br>
首先，请输入含有未知量的一元表达式，输入的内容中不要出现<font color = "red">空格</font>，否则会被识别为未知量而报错。</br>
而后请输入变量的赋值（无需输入变量名，系统会自动识别），并按回车键即可输出结果。</br>
本程序不支持多次输入，结果输出完成后就会退到模式选择栏。</br>
</a>

<a id = "function">

### 一元函数运算
<font color = "red" size = 3>注意：使用函数互调功能时，<font size = 4><b>必须</b></font> 要保证函数的变量名完全相同，否则会报错并判定为运行异常。</font></br>
要定义函数时，先输入“EDF”然后空格，再输入函数，格式为：f(x)=3+2*x，中间不要留空格，否则会判定为运行异常。</br>
要运行函数时，先输入“RUN”然后空格，再输入函数，格式为：f(2)并按回车即可。</br>
所有的运行中定义过程均不会报错，但运行时会检查函数的正确性。
本程序不支持同名函数，如果出现同名函数则会报错。</br>
</a>
